Renewable Performance Intern, Exus Renewables | June 2025 - July 2025

Programmed a tool to detect anomalies in wind and solar sites, utilizing Python and Pandas to pull, process and send data to an SQL server. Used Ignition to automate it so it runs daily and provides a list of problematic metrics to site managers. Estimated by head of operations to save Exus 6.8 million dollars annually.

In the summer of 2025 I worked for Exus Renewables as a renewable performance intern a second time and built a program to detect anomalies in wind and solar plants. This would dramatically increase the efficiency of the wind and solar plants as it would reduce the amount of cost of maintenance required on the solar plants, as by pinpointing anomalies early, Exus could perform preventative maintenance on them, requiring significantly less time and money spent repairing devices on the site.

To do this, I used python to pull down data from an API which I then stored in a postgres server. I then used python again to analyse the data in order to identify statistical anomalies for different metrics for the different devices, sorting them into anomalies and critical anomalies based on the degree to which they were different from the mean. I then sent this data into a PowerBI display which allowed anyone in the company to access it.

The end result of the project was a dashboard hosted on the internet, meaning that anyone in the company with proper credentials could access it. The dashboard was updated daily and processed 600,000 individual datapoints and allowed the user to see the number of anomalies on each site, which devices were causing the anomalies on each site, which metrics were responsible for the anomalies and how bad the anomalies were.

This project was a great success, as it was able to pinpoint exactly what was going wrong with each site, and allow both those running the plants and upper management to drill down into it in order to see exactly what was going wrong. The head of operations estimated it would save over 90,000 dollars a year per site, which resulted in estimated savings of 6.8 million dollars a year.

Renewable Performance Intern, Exus Renewables | June 2024 - July 2024

Developed documentation and a working API for a new SCADA system for wind and solar plants. Programmed three separate programs in Python with Pandas to pull down data and present it to site managers in PowerBI. Utilized these to pinpoint any plants or devices performing poorly and identify potential causes and solutions.

In the summer of 2024 I worked for Exus Renewables as a renewable performance intern and built a program to calculate how the wind turbines and solar panels were performing based on the input they were receiving. One issue with renewable energy is that since different places receive different amounts of sun or wind, underperforming devices can be masked by having higher inputs,and devices that are performing well, and whose setups should be emulated can be hidden by having worse inputs. As a result, having a comparable metric for all the turbines and all the solar panels across all the sites would be extremely useful for the company. To do this, I had to utilise an API to call down data for Exus. As the API was new to the company, a lot of my work was spent building out documentation for the company in order for them to be able to use the API properly. Then I built two programs using python that used the wind and solar power efficiency equations respectively by taking in the data from the turbines and solar panel, the meteorological data and latitude and longitude and height of the turbines via APIs. I then used power BI in order to create a dashboard for people to see it. The end result was a Power BI dashboard that showed how different wind turbines and solar panels across the sites had varied in performance across the years. This allowed us to track performance across all the sites in order to improve particular turbines or panels, and identify turbines and panels that were up to 30% less efficient than other panels due to easily fixable reasons. It also allowed us to track the degradation of turbines and solar panels from a performance in order to establish at what point replacements should be made to key components. This project was really successful as given that the company produces so much power, even a 10% increase in overall power production by bringing the under performing turbines up to standard would result in millions of extra dollars being made yearly across the company. It also allowed better enforcements of contracts, as the company could point to the turbines that were not up to standard and how quickly they fell out of standard in order to perform warranty claims.

Fifa Player Value Analysis | August 2025 - November 2025

Cleaned and prepared a dataset with more than 11 million datapoints for use with machine learning. Utilized PyTorch, Postgres and Spark on both local and Google Cloud compute to create multiple neural networks with greater than 90% accuracy at predicting the value of football players.

From August 2025 to November 2025, I worked on using machine learning to go through fifa player data in order to predict the ‘value’ of the fifa players based on their stats in the game. This was an exercise in building a pipeline that could take in unclean data, clean it and produce meaningful outputs - something that is necessary for all machine learning pipelines. By doing it with fifa data, we were able to access a lot of data and easily test how well the pipeline was performing. To do this, I built a pipeline using Pyspark, which took in the data and cleaned it for machine learning, using a postgres server to store the data. The data was then moved with parquet to PyTorch, where we trained a basic linear regression model, a shallow but wide model and a deep but narrow model, and assessed the difference between them. The end result was a pipeline that was able to handle the varied data of the fifa database while achieving greater than 80% accuracy with the shallow model and greater than 90% accuracy with the deep model. This project was very successful as our required max accuracy was only 85% so achieving greater than 90% proved that our data cleaning pipeline was performing exactly as intended.

Senior Mechanic for APEX | September 2021 - May 2025

Constructed 3 carbon fiber gravity racers over 4 years, implementing different carbon fiber layup and infusion techniques. Machined, designed and constructed steering and braking systems. Taught and guided new members on the construction process.

From August 2022 to May 2023, I was a senior mechanic in the construction of scorch. Scorch is a carbon fibre gravity racer that uses fork based steering, the first of its kind for the organisation making it. To design the buggy, we used solidworks to model the forces on both the buggy and the steering and then we then used carbon fibre layups in order to construct the shell of the buggy. Once the shell was constructed we then used cnc machines, lathes, waterjets and mills in order to construct the steering and the back axle. The buggy was very successful and won 3rd place in the design competition that year, despite having an entirely new steering system. It also has remained very durable - having lasted for 3 years requiring no meaningful fixes - with previous buggies sometimes lasting less than a year before having to be fully retired. It also performed faster than previous buggies - travelling down the freeroll 15% faster than any buggy constructed by apex in the last 5 years. As the first buggy of its type, the buggy was a great success in all metrics, and the success of it allowed other buggies to build off the baseline that it set. The buggy was expected to be a primarily experimental buggy and so the fact that it has lasted as long as represents a substantial achievement.
